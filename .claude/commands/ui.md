# /ui - Fresh UI 디자인

AI의 뻔한 디자인을 탈피하고 개성있는 UI를 생성합니다.

## 트리거

### 명령어
- `/ui`, `/fresh`, `/ui-design`

### 자연어 (레벨별)

| 레벨 | 트리거 키워드 |
|------|---------------|
| **common** | 기본, 평범하게, 무난하게, 안전하게, 기본적으로, simple, basic, safe, standard |
| **balance** | 적당히, 밸런스, 괜찮게, 보통으로, decent, balanced, moderate |
| **rare** | 독특하게, 개성있게, 남다르게, 특별하게, 차별화, unique, distinctive, different |
| **unique** | 실험적, 과감하게, 파격적, 미친듯이, 완전 새롭게, experimental, bold, crazy, wild |

### 트리거 패턴
```
"UI 만들어줘"                    → balance (기본값)
"독특한 UI로 만들어줘"            → rare
"로그인 페이지 실험적으로"         → unique
"랜딩페이지 기본적으로 만들어"      → common
"프론트 개성있게"                 → rare
"디자인 과감하게 해줘"            → unique
```

## 레벨 시스템

### 점수 매핑

| 레벨 | 안정성 | 개성 | 실험 | 설명 |
|------|--------|------|------|------|
| **common** | 90 | 30 | 10 | 검증된 패턴, 안전한 선택 |
| **balance** | 60 | 50 | 40 | 적당한 차별화, 무난한 개성 |
| **rare** | 40 | 80 | 60 | 흔한 패턴 회피, 눈에 띄는 |
| **unique** | 20 | 90 | 90 | 파격적, 예상 밖, 실험적 |

## 디자인 가이드

### 피해야 할 AI 패턴 (레벨에 따라 적용)

#### 색상
| 피할 것 | 대안 |
|---------|------|
| 파란 그라데이션 (#3B82F6 → #1D4ED8) | 단색, 무채색+포인트, 예상 밖 색조합 |
| 보라-핑크 그라데이션 | 모노톤, 보색 대비, 흙/자연색 |
| 회색 배경 (#F3F4F6, #F9FAFB) | 화이트, 딥블랙, 컬러 배경 |
| Tailwind 기본 팔레트 그대로 | 커스텀 색상, 채도/명도 조절 |

#### 레이아웃
| 피할 것 | 대안 |
|---------|------|
| 모든 것 중앙 정렬 | 비대칭, 좌측 정렬, 의도적 오프셋 |
| 균일한 카드 그리드 (3열) | 불규칙 그리드, 매거진 레이아웃, 오버랩 |
| Hero + 3-column features | 풀스크린 섹션, 스크롤 기반, 단일 포커스 |
| 반복적인 섹션 구조 | 리듬 변화, 예상 밖 배치 |

#### 타이포그래피
| 피할 것 | 대안 |
|---------|------|
| Inter, system-ui 기본 | 세리프, 특색있는 sans, 혼합 사용 |
| 16px / 1.5 line-height 균일 | 큰 대비 (12px vs 48px), 타이트한 행간 |
| font-medium 남발 | bold vs light 대비, 한 weight만 사용 |

#### 효과
| 피할 것 | 대안 |
|---------|------|
| shadow-lg 모든 카드에 | 노 섀도우, 보더만, 또는 극단적 섀도우 |
| rounded-2xl 균일 적용 | 날카로운 모서리, 혼합 (sharp + round) |
| hover:scale-105 | 색상 변화만, 또는 과감한 변형 |
| transition-all | 특정 속성만, 또는 즉각 반응 |

#### 컴포넌트
| 피할 것 | 대안 |
|---------|------|
| 카드 안에 아이콘 + 제목 + 설명 | 텍스트만, 또는 풀이미지, 또는 인터랙티브 |
| 둥근 프로필 이미지 | 사각형, 불규칙 마스크, 또는 없음 |
| 그라데이션 버튼 | 플랫, 아웃라인, 텍스트 링크 |
| 햄버거 메뉴 (모바일) | 항상 표시, 또는 완전히 다른 네비게이션 |

## 레벨별 적용 규칙

### common (안전)
- 피할 패턴: 상위 20%만 (가장 뻔한 것들)
- 대안 선택: 검증된 것 위주
- 목표: "AI가 만든 티"만 줄이기

### balance (기본값)
- 피할 패턴: 상위 50%
- 대안 선택: 적당히 신선한 것
- 목표: 눈에 띄되 안정적

### rare (독특)
- 피할 패턴: 상위 80%
- 대안 선택: 의외의 조합
- 목표: "이거 뭔가 다르다" 느낌

### unique (실험)
- 피할 패턴: 거의 전부
- 대안 선택: 파격적인 것 우선
- 목표: 예상을 깨는 디자인, 브루탈리즘/글리치/비대칭 등

## 스타일 힌트 (unique 레벨)

요청에 따라 특정 스타일 적용 가능:

| 스타일 | 특징 |
|--------|------|
| **브루탈리즘** | 날카로운 모서리, 강한 보더, 원색, 의도적 거칠기 |
| **미니멀 익스트림** | 극단적 여백, 단일 요소 포커스, 모노톤 |
| **레트로/Y2K** | 그라데이션 메시, 원색, 픽셀, 90-2000년대 느낌 |
| **글래스모피즘** | 반투명, 블러, 빛 반사 효과 |
| **네오브루탈** | 브루탈 + 네온/팝 컬러, 강한 그림자 |

## 실행 흐름

```
1. 트리거 감지 → 레벨 결정
2. 사용자 요청 분석 (어떤 UI를 만들지)
3. 해당 레벨의 "피할 패턴" 로드
4. 대안 선택하여 디자인
5. 코드 생성 (React/HTML/CSS)
```

## 출력 예시

### 요청: "로그인 페이지 독특하게 만들어줘"

```
[레벨: rare 감지]

적용된 차별화:
- 색상: 파란색 대신 모노톤 + 오렌지 포인트
- 레이아웃: 중앙 폼 대신 좌측 정렬 + 우측 일러스트
- 타이포: Inter 대신 세리프 헤딩 + 산세리프 본문
- 효과: 그림자 대신 1px 보더

[코드 생성]
```

## 주의사항

- 레벨은 트리거에서 자동 감지, 명시적 지정 가능
- 기본값은 balance (명시 없으면)
- 사용자가 특정 요소 지정하면 그것은 유지 (예: "파란색으로 해줘" → 파란색 유지)
- UX 기본 원칙은 준수 (접근성, 가독성 등)
