# 미니게임 플랫폼 개선 컨텍스트

## 참고 앱: 우리은행 WON PLAY

WON PLAY는 우리은행 앱 내 혜택 섹션에서 제공하는 미니게임 플랫폼으로,
사용자 참여를 유도하고 보상(꿀머니)을 제공하는 방식으로 운영됩니다.

---

## 현재 플랫폼 상태

### 구조
- Next.js 14 App Router 기반
- Supabase 백엔드 (랭킹 시스템)
- 총 13개 미니게임 (스도쿠, 2048, 메모리, 지뢰찾기, 워들, 슬라이딩퍼즐, 타이핑, 반응속도, 숫자야구, 플래피버드, 뱀, 벽돌깨기, 색상맞추기)

### UI 현황
- 단순 그리드 카드 레이아웃
- 이모지 기반 아이콘
- 기본적인 텍스트 정보만 제공
- 랭킹 페이지 별도 존재

### 1. 메인 화면 (게임 목록)

| 요소 | WON PLAY | 현재 플랫폼 | 개선 방향 |
|------|----------|-------------|-----------|
| 필터/카테고리 | 탭 필터 (미션, 생애주기, 편의, 인기) | 없음 | 카테고리 필터 추가 (퍼즐, 액션, 두뇌, 인기) |

| 섹션 구분 | "매일매일 WON PLAY" + 더보기 | 단일 목록 | 섹션별 구분 (오늘의 추천, 신규, 인기) |

| 요소 | WON PLAY | 현재 플랫폼 | 개선 방향 |

## 기술적 고려사항

### 이미지 에셋 관리
- 게임 썸네일: `/public/images/games/{game-id}/thumbnail.png`
- 게임 배너: `/public/images/games/{game-id}/banner.png`
- 아이콘: `/public/images/games/{game-id}/icon.svg`

### 컴포넌트 구조 (예상)
```
src/components/
├── home/
│   ├── GameList.tsx          # 게임 목록
│   ├── GameCard.tsx          # 게임 카드
│   ├── CategoryFilter.tsx    # 카테고리 필터
│   ├── HeroBanner.tsx        # 상단 배너
│   └── DailyMission.tsx      # 일일 미션 표시
├── game-detail/
│   ├── GameLobby.tsx         # 게임 시작 대기 화면
│   ├── GameBanner.tsx        # 게임 배너
│   ├── MiniRanking.tsx       # 미니 랭킹
│   └── MissionCard.tsx       # 미션 카드
└── mission/
    ├── DailyMissions.tsx     # 일일 미션 목록
    └── MissionProgress.tsx   # 미션 진행도
```

### 데이터 모델 확장 (예상)
```sql
-- 미션 테이블
CREATE TABLE missions (
  id SERIAL PRIMARY KEY,
  title TEXT NOT NULL,
  description TEXT,
  game_type TEXT,
  target_type TEXT,  -- 'score', 'play_count', 'win'
  target_value INT,
  reward_points INT,
  mission_type TEXT, -- 'daily', 'weekly', 'achievement'
  created_at TIMESTAMP DEFAULT NOW()
);

-- 사용자 미션 진행
CREATE TABLE user_missions (
  id SERIAL PRIMARY KEY,
  user_id TEXT REFERENCES users(id),
  mission_id INT REFERENCES missions(id),
  progress INT DEFAULT 0,
  completed_at TIMESTAMP,
  created_at TIMESTAMP DEFAULT NOW()
);

-- 포인트 내역
CREATE TABLE point_history (
  id SERIAL PRIMARY KEY,
  user_id TEXT REFERENCES users(id),
  amount INT,
  reason TEXT,
  created_at TIMESTAMP DEFAULT NOW()
);
```
